我的第一篇博客文章

架设个人Blog

这是我的第一篇博客，主要记录一下在开源软件开发课程中进行个人Blog架设中遇到的问题以及如何解决。

遇到的问题（闲谈版）

其实在架设个人Blog的前面都很顺利，这让我非常激动，心里还幻想和渗透测试一样的，各个地方突然出现的各种不知道什么原因的问题不会在架设Blog中出现了。事实证明我还是高兴太早了。等我配置完hexo，更换完主题后，开始撰写我的博客文章，按照PPT我打算在本地预览一下。运行完命令后成功跳出了一个链接，点进去，结果是乱码。

尝试解决问题

不知道为何，开始询问AI，AI给出了一长串可能的问题，可能是内容滞留，可能是没有安装渲染器，可能是node.js版本不是最新版本。我一个一个试了。重新安装了渲染器，没解决，删除滞留内存，没解决，删除bolg文件夹重新安装，没解决，删除node.js重新安装，没解决。经过多次重试，我发现就是安装主题之后开始出现乱码的，但我还是不知道哪里出问题了，遂决定先不安装主题了，先写blog。

配置SHH密钥

添加SSH公钥问题

将SSH公钥添加到Git Hub的时候以为是将路径复制到New SSH key中的文本框，所以按添加后跳出了 无法找到我的公钥。后面经过询问AI进行多个途径排查，发现原因是：那个路径是密钥存放的地址，不是密钥，前往地址找到密钥后成功添加。

测试SSH连接问题

回到 Git Bash 窗口，输入命令：ssh -T git@github.com之后，回答是无法连接。遂又拍照寻找AI，经过一系列排查，最后尝试使用HTTPS端口（443）
echo “Host github.com
Hostname ssh.github.com
Port 443” >> ~/.ssh/config
ssh -T git@github.com
执行此命令后成功连接。

部署博客问题

打开 hexo 的配置文件 _config.yml 修改 deploy 位置的配置 之后，在博客目录下运行hexo cl && hexo g && hexo d 结果报错说找不到git。在询问AI进行一系列排查各种可能之后，发现是静态网站生成工具的配值问题，运行
npm install hexo-deployer-git –save 命令之后，成功找到git，完成博客部署。
